📋 OBJETIVO ALCANÇADO:
────────────────────────────────────────────────────────────────────────────────
A aplicação foi modificada para aceitar, processar e gerenciar produtos com
6 colunas de preço (antes eram 4), incluindo preços de distribuidor e revenda,
com e sem IPI.

📊 NOVO FORMATO CSV:
────────────────────────────────────────────────────────────────────────────────
Fabricante;Descrição;Preço Distribuidor s/ IPI;Preço Distribuidor c/ IPI;Preço Unit. Final s/ IPI;Preço Final c/ IPI

Exemplo:
5L500;BATERIA DE LITIO NAO RECARREGAVEL;2.288,58;2.511,72;2.542,87;2.790,80

✅ Coluna 1: Código do produto (fabricante)
✅ Coluna 2: Descrição do produto
✅ Coluna 3: Preço Distribuidor SEM IPI
✅ Coluna 4: Preço Distribuidor COM IPI
✅ Coluna 5: Preço Final/Revenda SEM IPI
✅ Coluna 6: Preço Final/Revenda COM IPI

🗄️ ESTRUTURA DO BANCO DE DADOS:
────────────────────────────────────────────────────────────────────────────────

Tabela: products
├── id (UUID) - Identificador único
├── code (VARCHAR) - Código do produto (único)
├── description (TEXT) - Descrição
├── marca (VARCHAR) - Fabricante
├── price_distributor (DECIMAL) - Preço distribuidor s/ IPI
├── price_distributor_with_ipi (DECIMAL) - Preço distribuidor c/ IPI
├── price_final (DECIMAL) - Preço final s/ IPI
├── price_final_with_ipi (DECIMAL) - Preço final c/ IPI
├── priceResale (DECIMAL) - Compatibilidade
├── priceResaleWithIPI (DECIMAL) - Compatibilidade
├── created_at (TIMESTAMP) - Data de criação
└── updated_at (TIMESTAMP) - Data de atualização

🚀 COMO USAR:
────────────────────────────────────────────────────────────────────────────────

PASSO 1: Preparar Banco de Dados
   → Abra: EXECUTE_ESTE_SQL.md
   → Copie TODO o SQL
   → Cole no SQL Editor do Supabase
   → Clique em RUN

PASSO 2: Importar CSV
   → Aplicação → ⚙️ Configurações
   → Clique em "Importar CSV" (Gerenciar Produtos)
   → Selecione seu arquivo CSV com 6 colunas
   → Aguarde confirmação

PASSO 3: Verificar
   → Você verá: "Total de produtos: XXX"
   → Pronto! ✅

✅ VALIDAÇÕES APLICADAS:
────────────────────────────────────────────────────────────────────────────────

✓ Mínimo 2 linhas (1 cabeçalho + 1 produto)
✓ Exatamente 6 colunas em cada linha
✓ Preços são números válidos (conversão de , para .)
✓ Código e descrição não vazios
✓ Sem códigos duplicados
✓ Preços ≥ 0 (validação no banco)
✓ Separador correto (;)
✓ Decimal correto (,)

⚠️ IMPORTANTE:
────────────────────────────────────────────────────────────────────────────────

• O comando DROP TABLE deleta TODOS os dados sem recuperação
• Faça backup de dados antigos antes de executar
• Se quer manter dados antigos, remova a linha DROP TABLE do SQL
• A tabela será criada do zero com a estrutura correta
